# RiotBus MVP Page Spec

## 1. 文档目的

本文档用于把 `RiotBus` 的 MVP 页面规则写清楚，方便后续进入 Figma 高保真、前端开发、接口设计和测试验收。

本文不解决视觉风格细节，重点确认：
- 页面有哪些模块
- 用户点了什么会发生什么
- 什么状态下按钮可用
- 数据和 AI 内容如何展示
- 异常情况如何提示

当前实现已经进入可运行 MVP 状态。本文档保留页面规则，同时记录当前实现口径，后续改版优先先改本文档再改代码。

## 2. 页面清单

MVP 包含：
- 首页
- 资讯详情弹窗
- 筛选弹窗
- 对比结果页
- PDF / 分享弹窗

MVP 不包含：
- 独立艺人详情页
- 独立专题页
- 用户登录系统
- 用户历史记录页

## 3. 首页

### 3.1 页面目标

首页负责三件事：
- 展示 `RiotBus / 乱斗巴士` 品牌
- 用轻资讯 Banner 营造吃瓜氛围
- 让用户选择模式并进入 PK 流程

### 3.2 顶部品牌区

展示内容：
- `RiotBus`
- 可选中文副标题：`乱斗巴士`

规则：
- 品牌名在首屏必须明显
- 不需要放太多解释文案

### 3.3 News Banner 轮播

Banner 数量：
- 默认展示 `3` 条 news
- 中间卡片为当前主 news
- 左右两侧为上一条 / 下一条 news

切换方式：
- 用户点击下方切换条切换
- 可支持自动切换

动画方向：
- 切换到下一条时，中间 Banner 向左或向右移动
- 两侧 Banner 跟随进入主位
- 动画要有“卡片换位”的感觉

内容维护：
- Banner 内容由网站运营者维护
- 每条 Banner 包含图片、标题、正文、来源链接
- MVP 不需要做复杂新闻字段系统，先按运营内容维护即可

点击行为：
- 点击当前 Banner 打开资讯详情弹窗
- 点击左右侧 Banner 可切换到对应 Banner

### 3.4 模式介绍区

模式数量：
- `Mean / 刻薄女孩`
- `Neutral / 清清白白`

展示规则：
- 页面中展示两种模式的简短介绍
- 当前文案可先作为占位，后续再统一润色

文案方向：
- `刻薄女孩`：刻薄、有梗、可能使用黑话和争议素材
- `清清白白`：客观、无黑料、适合快速了解

当前实现：
- `刻薄女孩` 使用绿色主题、mean 头像、mean 显示名和 RAG 黑话语料。
- `清清白白` 使用粉色主题、neutral 头像、neutral 显示名和中立输出约束。

### 3.5 模式切换条

交互：
- 用户点击 `Mean / 刻薄女孩` 或 `Neutral / 清清白白` 切换模式
- 当前选中项高亮
- 默认可选 `Mean`，后续可根据产品调性再定

确认按钮：
- 点击 `OK` 打开筛选弹窗
- 打开弹窗时将当前模式带入

### 3.6 首批 Banner 运营内容示例

以下内容先作为首页 Banner 的运营示例，正式上线前需要补来源链接并做事实核验。

Banner 1：
- 标题：Lady Gaga 因身体不适取消演唱会
- 方向：演出临近取消，引发粉丝和路人讨论
- 内容感：偏事件说明 + 舆论反应

Banner 2：
- 标题：Olivia Rodrigo 新单回归冲榜，官方账号下场求买单
- 方向：`dropdead` 对撞 `Choosing Texas`，官方宣传动作拉满
- 内容感：偏冲榜战况 + 粉圈动员

Banner 3：
- 标题：Ariana Grande 7.31 回归，AG8 `Petal` 要来了
- 方向：回归预告、专辑期待、粉丝情绪拉满
- 内容感：偏爆炸预告 + 热点上车

## 4. 资讯详情弹窗

### 4.1 触发方式

用户点击当前主 Banner 后打开。

### 4.2 弹窗结构

左侧：
- 图片
- 图片下方 source 来源链接

右侧：
- 标题
- 简短正文
- 关闭按钮

### 4.3 内容规则

内容由运营者维护。

当前 Banner 图片统一放在：

```text
public/assets/banners/
```

命名：

```text
gaga-cancel.png
olivia-dropdead.png
ariana-petal.png
```

写法方向：
- 字少
- 图简
- 有冲突点
- 有吃瓜感
- 不直接把未核实内容写成事实

### 4.4 交互规则

- 点击关闭按钮关闭弹窗
- 点击来源链接打开对应来源
- 弹窗不提供强制 PK 入口

## 5. 筛选弹窗

### 5.1 触发方式

用户在首页选择模式后，点击 `OK` 打开。

### 5.2 当前模式展示

弹窗左上角展示当前模式，例如：
- `刻薄女孩 Mode`
- `清清白白 Mode`

规则：
- 弹窗不重复大段解释模式含义
- 仅展示当前模式作为用户确认

### 5.3 艺人选择

选择方式：
- 用户单击艺人头像圆圈选择艺人
- 选中后右上角显示勾选状态

数量限制：
- 第一版只允许选择 `2` 位艺人
- 未选满 `2` 位时，确认按钮不可用
- 选满后展示 `已选 2/2`

状态展示：
- `已选 0/2`
- `已选 1/2`
- `已选 2/2`

### 5.4 维度选择

维度包含：
- 销量：CM
- 流媒体：Spotify
- 奖项：GRAMMY
- 乐评：AOTY / RYM

选择方式：
- 用户单击维度右侧方框选择
- 已选中时方框显示高亮或勾选状态

规则：
- 至少选择 `1` 个维度
- 未选择任何维度时，确认按钮不可用

### 5.5 AI API 配置

支持两种方式：
- 使用平台 default AI
- 用户填写自己的模型地址和 API Key

自定义字段：
- `base_url`
- `api_key`
- `model`

校验规则：
- `base_url` 和 `api_key` 都填写后，用户可点击 `check`
- 校验成功后允许继续
- 校验失败时显示小弹窗报错

默认 AI：
- MVP 提供平台免费 default
- 用户不填写自定义 API 时可直接使用 default
- default AI 免费开放，直到平台额度用完
- MVP 阶段先不做复杂付费策略
- 后续如出现滥用，再增加限流、排队或关闭 default

### 5.6 确认按钮可用条件

确认按钮可用需要满足：
- 已选择 `2` 位艺人
- 至少选择 `1` 个维度
- 使用 default AI，或自定义 API 校验成功

点击确认后：
- 关闭筛选弹窗
- 跳转到对比结果页
- 带入当前模式、艺人、维度、AI 配置

## 6. 对比结果页

### 6.1 页面目标

对比结果页负责展示：
- 左侧结构化数据
- 右侧 AI 智能体报告和追问

### 6.2 顶部信息

展示：
- 当前模式
- 艺人 1
- 艺人 2

### 6.3 左侧数据表

表格列：
- 维度
- 艺人 1
- 艺人 2

默认维度顺序：
1. 销量
2. 流媒体
3. 奖项
4. 乐评
5. 链接

生成规则：
- 根据用户选择的维度生成对应行数
- 如果用户全选，则展示四个数据维度 + 一个链接区域
- 如果某维度未选择，则不展示该行
- 链接区域始终展示，用于集中放来源链接

缺失状态：
- 如果某个数据缺失，显示 `缺失`
- AI 报告中也需要说明对应维度缺失

### 6.3.1 维度字段 v0

MVP 的数据表保持极简，每个维度只展示最适合横向比较的核心值。

销量：CM
- 展示值：艺人页面总销量 / 总单位
- 示例：`120M`
- 数据方式：CM 艺人页采集或手工补录

流媒体：Spotify
- 展示值：Spotify followers
- 示例：`110M followers`
- 数据方式：ChartMasters Spotify followers 总榜人工补录
- 说明：MVP 不做月听众；流媒体人气统一看 followers

奖项：GRAMMY
- 展示值：总获奖 / 总提名 / 通类获奖数
- 示例：`14 wins / 52 noms / 3 GF`
- 通类范围：年专、年制、年歌、年新
- 口径：`GF` 只统计通类获奖，不把通类提名算进去；如果后续要展示通类提名，需要单独加 `GF noms` 字段

乐评：AOTY / RYM
- 展示值：AOTY 艺人主页 critic / user
- 示例：`80 / 77`
- 数据方式：AOTY 艺人页采集
- RYM 作为补充来源，MVP 可先放入链接区或后续加入

链接：
- Spotify 艺人页
- GRAMMY 艺人页或搜索页
- AOTY 艺人主页
- RYM 艺人主页
- CM 来源说明或内部补录说明

字段展示原则：
- 表格行按照用户选择的维度生成
- 默认排序为 `销量 -> 流媒体 -> 奖项 -> 乐评 -> 链接`
- 每个维度优先只展示一个核心对比值
- 不在每个数据格里塞 source，来源统一放到链接区域

### 6.4 来源链接区

来源链接统一放在表格最后一块。

展示内容：
- Spotify 艺人页
- GRAMMY 艺人页或相关页面
- AOTY 页面
- RYM 页面
- CM 来源说明

规则：
- 每条链接标注来源类型
- 如果没有公开链接，显示 `内部补录 / 待补来源`
- 不要求每个数据格单独放 source

## 7. AI 智能体区

### 7.1 首次生成

进入对比结果页后，AI 主动生成首轮报告。

报告内容：
- 两位艺人的核心对比
- 用户所选维度的优劣势
- 数据缺失说明
- 当前模式对应的语气表达

### 7.2 模式语气

`Mean / 刻薄女孩`：
- 语气更刻薄
- 可使用黑话表、粉圈语料、争议事件
- 允许有 mean 味，但不能凭空造谣

`Neutral / 清清白白`：
- 客观
- 不使用黑料
- 不主动挑事
- 更像入门科普和数据总结

### 7.3 用户追问

用户可继续提问、反驳或要求补充。

规则：
- AI 回复沿用当前模式语气
- AI 必须围绕当前两位艺人和当前上下文
- 不应跳出当前 PK 随意扩展

### 7.4 黑话表第一版范围

第一版黑话表覆盖首发 `20` 位艺人。

内容范围：
- 艺人外号
- 常见粉圈梗
- 常见拉踩话术
- 公开争议事件摘要
- 不同艺人之间常见对线角度

使用规则：
- 只在 `刻薄女孩` 模式使用
- `清清白白` 模式不调用黑话表
- 黑话表内容需要分风险等级
- 高风险内容只作为语气参考，不直接原样输出

## 8. PDF / 分享弹窗

### 8.1 触发方式

用户在结果页点击生成报告 / 分享按钮后打开。

### 8.2 生成逻辑

系统调用 AI 总结当前 PK 内容。

输入包含：
- 当前结构化数据
- AI 首轮报告
- 用户后续追问和 AI 回复
- 当前模式
- 黑话表或中立模板

输出包含：
- AI 生成标题
- PK 总结正文
- 可用于分享的 PDF / 卡片内容

### 8.3 标题规则

`Mean / 刻薄女孩` 模式：
- 可结合黑话表生成更有趣、更 mean 的标题
- 标题要有梗，但不能写成无依据指控

`Neutral / 清清白白` 模式：
- 标题客观清楚
- 直接表达对比主题

### 8.4 内容规则

正文包含：
- 对比摘要
- 关键优势
- 关键短板
- 数据缺失说明
- 综合对话内容后的结论

### 8.5 背景样式

用户可切换 PDF / 分享卡背景。

MVP 背景可先提供：
- 默认浅色
- 强对比色
- 霓虹感
- 深色

### 8.6 导出

导出格式：
- PDF

后续可扩展：
- PNG 分享图
- 长图

## 9. MVP 状态清单

必须处理的状态：
- Banner 切换中
- 资讯弹窗打开 / 关闭
- 艺人未选满
- 艺人已选满
- 维度未选择
- 自定义 API 校验中
- 自定义 API 校验失败
- 自定义 API 校验成功
- 数据加载中
- 数据缺失
- AI 生成中
- PDF 生成中
- PDF 生成失败
